    ℹ️ DEVELOPER NOTES — Read Before Going Live
    Q: Where is data saved?
    Currently nowhere. This is a standalone HTML file. On submit it only generates a local Reference ID in the browser. No data is stored or sent anywhere.

    Q: Is this form live?
    Not yet. It is a static HTML file. It needs a backend to become live.

    Q: How to make it live — 3 options:
    Option A — Simplest: Google Forms + Sheets
    Use Google Forms for data collection. Free, no coding needed. Data auto-saves to Google Sheets. Limitation: cannot match this form's design.
    Option B — Recommended: Google Sheets backend (minimal code)
    1. Create a Google Sheet for registrations.
    2. Go to Extensions → Apps Script → deploy as Web App.
    3. Use this endpoint URL in the form's submit function (replace the submitEnquiry() call).
    4. Host the HTML file on GitHub Pages (free) or any web host.
    5. Total cost: ₹0. Setup time: ~2 hours.
    Option C — Full stack (recommended for scale)
    Backend: Node.js + Express or Firebase.
    Database: Firestore (Firebase) or MongoDB Atlas — free tier sufficient for first 1,000 members.
    Hosting: Netlify / Vercel (free tier).
    This gives you a full dashboard, SMS triggers, auto WhatsApp messages (via Wati/Interakt).
    Q: Payment Gateway — Razorpay
    Razorpay is pre-wired in this form. To activate:
    1. Sign up at dashboard.razorpay.com (Indian company, instant KYC).
    2. Get your Key ID from Settings → API Keys.
    3. Replace rzp_test_REPLACE_WITH_YOUR_KEY in the JS with your live key.
    4. Add the Razorpay checkout script: <script src="https://checkout.razorpay.com/v1/checkout.js"></script>
    5. For production, create orders server-side (required by Razorpay for security).
    Razorpay fees: 2% per transaction (no setup cost, no monthly fee). Settlement next business day.
